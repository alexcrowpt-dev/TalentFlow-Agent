# Contributing to TalentFlow Agent

We welcome contributions from the community! By participating in this project, you agree to abide by our code of conduct.

## üí° How to Contribute

### Reporting Bugs
1.  Check the existing issues to see if the bug has already been reported.
2.  If not, open a new issue with a clear title and detailed description.
3.  Include steps to reproduce the bug, expected behavior, and actual behavior.

### Suggesting Enhancements
1.  Open an issue to discuss the feature or enhancement you would like to propose.
2.  Clearly describe the problem the enhancement solves and how it aligns with the project's vision (see [README.md](README.md)).

### Code Contributions
1.  **Fork** the repository.
2.  **Clone** your fork locally.
3.  Create a new branch for your feature or fix: `git checkout -b feature/my-new-feature` or `git checkout -b fix/bug-description`.
4.  Ensure your code adheres to the project's coding standards (Python: Black, Flake8; TypeScript: ESLint).
5.  Write clear, concise commit messages.
6.  **Test** your changes thoroughly.
7.  Submit a **Pull Request (PR)** to the `main` branch of the original repository.
